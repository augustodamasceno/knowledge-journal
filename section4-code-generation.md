## The Core Format: `AGENTS.md`

`AGENTS.md` is a simple open format for guiding coding agents — think of it as a README for agents. It is stewarded by the Agentic AI Foundation under the Linux Foundation. The canonical reference is [agents.md](https://agents.md).

`AGENTS.md` complements `README.md`: while README targets human developers, `AGENTS.md` contains technical instructions specifically for AI coding agents — exact commands, code patterns, testing strategies, and project-specific gotchas. It works across OpenAI Codex, GitHub Copilot, Cursor, Claude Code, Aider, and more.

## Key Structural Patterns

### Hierarchical layout (recommended by HumanLayer)

Keep task-specific instructions in separate files under `agent_docs/`:

```
agent_docs/
├── building_the_project.md
├── running_tests.md
├── code_conventions.md
├── service_architecture.md
├── database_schema.md
└── service_communication_patterns.md
```

The root `CLAUDE.md` lists them with descriptions and instructs the agent to decide which are relevant before starting work.

### Scope-aware instructions (GitHub Copilot style)

Copilot supports scoped instructions via `.github/instructions/*.instructions.md` files with glob-pattern frontmatter, letting you define different rules for TypeScript vs Python, or frontend vs backend, without polluting a single file.

```
.github/
├── copilot-instructions.md        # repository-wide
└── instructions/
    ├── react-components.instructions.md   # applyTo: "**/*.tsx"
    └── api-routes.instructions.md         # applyTo: "src/api/**"
```

### Keep it lean

Claude Code's system prompt already contains ~50 individual instructions before your own rules are loaded. This means your `CLAUDE.md` should contain as few instructions as possible — only those universally applicable to every session.

## Multi-tool Setup

The cleanest multi-tool setup: one `AGENTS.md` as the source of truth, with symlinks pointing the tool-specific filenames at it.

```bash
ln -s AGENTS.md CLAUDE.md
ln -s AGENTS.md .github/copilot-instructions.md
ln -s AGENTS.md .cursorrules
```
